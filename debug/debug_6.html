  1. 2주차 Intro

 

드디어 함수를 만나게 되었다! 또한 클래스를 다루면서 객체와 인스턴스 등 반가운 용어들을 만나게 되었다. 이후로는 파일 입출력, try-except 구문, streamlit을 거쳐 Git 사용법을 다루었다. 나는 예전에 Matlab에서 직접 데이터를 돌리거나, matplotlib를 다루었었기 때문에 이번 주차에서 다룬 전반적인 내용들은 낯선 환경이 아니었지만, 아무래도 streamlit은 처음 사용하는 것이다보니 Markup 언어를 다루던 때가 떠올라서 기분이 영 좋지 않았다.(나는 프론트엔드에 그닥 재미를 느껴본 적이 없다....) 게다가 Git 역시 레포지토리만 만들어본 정도이기에 새로운 배움은 신기했다. Git bash까지 설치하여 branch를 생성함으로써 팀프로젝트 진행에 대한 입출력을 다루었는데, Git bash가 리눅스 기반이었기 때문에 Putty를 쓰던 때가 떠올라서 수업을 재밌게 들었다. 안그래도 리눅스에서 ls 명령어로 경로를 확인하다가 윈도우에서 dir 명령어가 기억이 안나서 버벅대는게 당장 작금의 웃픈 현실이었기에.... 여튼, 이번 주차에 대한 나의 대답은 다음과 같다.

 

'반갑다, 리눅스 명령어들아!'

 

 

 

  2. 인스턴스와 객체, 그리고 클래스에 대하여

 

클래스를 다루다보면 객체와 인스턴스를 필연적으로 짚고 넘어가야 한다. 물론 객체와 인스턴스에 대한 관용적인 비유로 붕어빵 틀과 붕어빵의 관계라고 설명을 하지만, 굉장히 레거시하기도 하고 초심자 입장에서는 와닿지 않는다고 생각한다. 다행이도 프로그래밍 언어는 개념적인 측면에서 1대 1 대응이 되는 관계가 아주 많다. 가령, 주소값과 포인터, 레퍼런스와 밸류, 절차지향과 객체지향 등이 바로 그것이다. 물론 처음 접하는 입장에서는 어려우리라 생각한다. 나 역시도 비유를 드는 것을 별로 좋아하지 않고, 정의된 개념을 있는 그대로 받아들이는 것이 가장 좋은 방법이라고 생각한다. 하지만 사람마다 생각하는게 다 다르듯이, 나는 내 방식대로 이 개념을 큰 틀에서 비유로 설명하고자 한다.

 

머릿속에 한 명의 인간이 있다고 가정해보자. 우리는 해당 인간의 신체를 손, 손목, 팔, 어깨, 가슴, 목, 머리, 허리 등 여러 부분들로 나눠서 설명이 가능하다. 그런데 우리가 해당 신체부위가 꼭 그 신체부위가 맞는지 뜯어서 확인하지는 않는다. 겉으로 드러난 부분만 보고도 어느 신체부위인지 확인이 가능하다. 그럼 그 신체부위를 다 모으면 인간이 되는가? 그것도 아니다. 적절한 배치를 맞춰야 한다. 그럼 적절한 배치를 맞춘다고 인간이 움직이는가? 그것도 아니다. 혈액이 들어가서 인간의 신체를 돌아다녀야 인간이 움직일 수 있다. 자, 이 설명을 이해한다면 우리는 객체와 인스턴스, 그리고 클래스에 대해서 다 이해한 것이다!

 

앞선 비유들을 Top-down으로 내려가며 이해해보자. 클래스를 인간이라고 가정할 때, 클래스가 나뉘면 객체가 된다. 인스턴스는 객체가 소프트웨어 상에서 실체화 된 것을 말한다. 즉, 표면상으로 드러나있기에 우리가 주로 다루는 값들이라고 생각하면 편하다. 또한 해당 클래스가 적절하게 작동할 수 있도록 초기값으로 생성자-나는 생성자란 말이 직관적이지 않아서 그닥 좋아하지 않는다-를 선언한다. 이제는 Bottom-up으로 올라가며 이해해보자. 초기값으로 세팅한 생성자가 적절한 순서로 배치된 객체를 돌고, 적절한 순서로 배치된 객체가 모여 클래스가 된다. 그렇게 클래스에서 나온 인스턴스를 주로 다루게 되는 것이다!

 

 


  3. 2주차 코드에 대해 다시보자

 

  - 함수

함수와 반환의 개념에 대해서 먼저 설명하자면, 왼쪽에 위치한 공간에 = 함수(파라미터)를 할당하는 형태이다. 파라미터는 함수를 움직이는 매개변수이며, 아규먼트는 함수를 호출할 때 실제로 함수에 전달되는 값이다. 즉, 아규먼트를 넣어서 함수를 호출하면 아규먼트의 값을 파라미터(매개변수)에 할당한다라고 생각해도 좋겠다.

 

자, 설명이 끝났으니 다시 위의 코드로 넘어오자면, 두 가지 parameter에 넣어줄 argument들을 변수로 선언해서 동작시키는 모습이다. 파라미터를 거쳐서 값을 넣어주는 경우는 직접 맵핑이 되기에 순서가 달라도 상관없다. 다만, 모든 값들을 순서에 맞춰서 파라미터를 거치지 않고 넣어준다면 모를까, 파라미터에 거치는 경우와 직접 넣어주는 경우를 섞는다면 해당 함수는 작동되지 않는다.

 

  - 패키징과 언패키징

다중 입력은 패키징으로 묶어서 처리할 수 있다. 위의 코드는 end를 써서 한 줄에 모두 나온 것이지만, 실제로는 한 줄씩 출력이 된다. 아래의 코드는 5개의 파라미터를 입력받는다. 따라서 입력값은 무조건 5개가 되어야 작동을 한다.

 

  - 람다

람다 함수는 연산의 일종이라고 생각하면 편하다. 일회성으로 사용되기에 자주 나오는 것은 아니지만, 알면 편하게 쓸 수 있다. map 함수는 여러 개의 요소들을 지정된 함수에 대한 결과물로 각각 처리를 하여 새 결과로 생성해주는 함수이다.

 

  - 랜덤 초이스

내장 모듈인 랜덤을 불러와서 패키징을 하여 난수를 반환하는 코드이다. 일상 생활에서 불편함을 느껴 콘솔창에서 pyinstaller를 사용하여 콘솔형 프로그램을 만들어본 사람들이라면 익숙한 광경일 것이다.(그렇다고 프로그램을 다른 환경에서 사용가능하도록 전달하는건 다른 문제이긴 하다.)

 

  - 업무 성과

강사님께서 좋은 문제를 내주셨다. 다중입력 및 다중반환과 연관된 내용이다. 문제의 난이도 자체는 쉽다.

 

  - 월급 계산 및 세금 공제

이 문제도 좋은 문제라고 생각하여 가져왔다. 다중입력에 대한 다중반환값을 함수 밖에서 선언 및 처리하는 문제이다.

 

  - 클래스: 클래스 속성, 생성자, 그리고 메소드

클래스에 대한 기본 내용이다. 클래스는 전역변수로 지정되는 클래스 속성, 초기값으로 작동되는 생성자, 해당 클래스 전체가 공유하며 클래스 밖에서 활용할 수 있는 메소드로 나뉜다.

 

  - 클래스 속성과 인스턴스 속성의 차이

빈 리스트를 전역변수로 생성 후, append라는 메소드를 add_skill로 선언하여 사용할 수 있다. 이 경우, append 메소드를 직접적으로 사용하는 것은 불가능하다.

 

  - 클래스 속성과 인스턴스 속성의 우선순위

클래스 속성과 인스턴스 속성이 충돌할 경우, 인스턴스 속성이 우선시된다. 이 경우, __class__를 통하여 원래 클래스의 속성이 무엇인지 접근할 수 있다.

 

  - 네임스페이스 테스트: 전역과 로컬

전역변수를 global 명령어를 통해 로컬 공간으로 가져와서 변경할 수 있다.

 

  - 네임스페이스 테스트: 논로컬

nonlocal 명령어는 전역변수가 아닌 바깥함수에 있는 변수를 해당 로컬공간으로 가져와서 변경할 수 있다.

 

  - 상속

상속이란 상위 클래스의 인자들을 하위 클래스의 인자들인 것처럼 사용할 수 있는 것을 말한다.

 

  - 다중 상속

다중 상속이란, 상위 클래스들의 인자들을 하위 클래스의 인자들인 것처럼 사용할 수 있는 것을 말한다.

 

  - 오버라이딩

오버라이딩이란, 상위 클래스들의 인자들을 하위 클래스로 가져와 덮어쓰는것을 말한다. 클래스가 상속구조일 경우, 위와 같이 __mro__ 명령어를 통해서 클래스의 다중상속구조를 파악할 수 있다.

 

  - 상속의 특성 통합 테스트

좋은 문제라고 생각하여 가져와봤다. 하위 클래스 입력값의 갯수는 상위 클래스 입력값의 갯수에 영향을 받는다. 다만, 반환되는 모든 값들을 사용할 필요는 없다.  

 

  - import와 from-import

import 명령어만 사용하여 호출할 경우, 함수명과 변수명에 모듈명을 전부 붙여줘야 한다. 매우 번거롭다.

이 경우에 as 명령어를 통해 모듈명을 축약해서 사용할 수 있다. 앞으로 많이 보게 될 광경일 것이다.

from-import 명령어로 불러올 수도 있겠지만, 이 역시 번거롭긴 매한가지다.

참고로 패키지 내의 다른 모듈을 불러오기 위해선 절대경로로 설정해줘야 한다....

 

  - 내장모듈함수 math

내장모듈 math를 불러와서 올림에 해당되는 함수인 ceil과 내림에 해당되는 함수인 floor를 사용할 수 있다. round는 내장모듈함수가 아닌 내장함수라서 자체적으로 사용이 가능하다.

 

  - 모듈함수 datetime

datetime의 경우, datetime 모듈 안에 datetime이라는 클래스가 있는 형태이다. 따라서 from-import로 두 번 선언해주거나, import로 호출할 경우, 함수 앞에 datetime.datetime. 형태로 붙여써야 한다.

 

  - 파일 입출력: 쓰기

write 모드일 때 파일이 없는 경우, 해당 파일은 없어도 되지만 디렉토리는 반드시 있어야 한다.

 

  - 파일 입출력: 읽기

반면에 read 모드일 때 파일이 없는 경우, 아예 에러가 발생한다.

 

  - csv 파일 생성 및 입출력

writerow 명령어를 사용하면 리스트 형태로 한 줄씩 입력이 가능하다.

 

  - log 파일 읽기 및 조건부 출력

파일을 읽고, writelines 명령어를 이용하여 다중 줄입력이 가능하다. for문을 사용하면 특정 명령어가 있는 구문만 호출이 가능하다.

 

  - try-except 1

이것도 프로젝트를 진행하다보면 많이 볼 광경인 try-except 구문이다. except는 if문에서 if문에 elif 및 else가 종속되어있는것과 마찬가지로, try문에는 except와 finally가 종속되어 있다.

 

  - try-except 2

다만 이 경우에 순서가 중요한데, elif 다음에 else가 오는 것과 마찬가지로, except 다음에 finally가 와야 한다. else는 예외가 발생하지 않았을 경우에만 실행이 가능하도록 섞어쓸 수 있다.

 

  - try-except 3

이렇게 출력구문으로 입력값에 대한 오류가 어떻게 발생했는지 알려주는 것도 가능하다. 여기서도 마찬가지로 else 구문을 사용했는데, else 구문은 의도적인 오류 발생이 필요할 때 쓰는 구문으로도 이용이 가능하다. else 구문 하위에 raise '출력하고싶은오류구문('오류시 출력구문')의 형식으로도 만들 수 있다.

 

  - streamlit

콘솔창에서 pip install streamlit으로 설치하여 사용할 수 있으며, 실행 명령어는 streamlit run '파일이름.py'이다. title, header로 유츄할 수 있겠지만 프론트엔드처럼 동작한다.

빈 컬럼값을 입력함으로써, 해당 페이지의 여백을 주는 것도 가능하다. 

data_editor()를 사용하면 실제 화면상에서 변경되는 모습을 볼 수 있다.

다만 프론트엔드'처럼' 동작한다고 했지, 실제 프론트엔드가 아니다. streamlit은 화면을 생성할 때마다 처음부터 코드를 렌더링하는 방식이라 저장이 되지 않기 때문에, count를 하고자하는 경우에는 초기 설정을 잡아줘야만 하는 번거로움이 있다. 

나는 streamlit을 처음 써봐서 몰랐던 사실인데, 위와 같이 서브페이지의 경우 디렉토리명을 pages로 고정해야 한다. 또한, _를 공백으로 인식하기 때문에, 페이지의 순서를 설정하고자 하면 페이지명 앞에 '숫자_' 형태로 만들어두는 방법이 있다.

 

  - Git bash 명령어

리눅스 기반의 명령어로 동작하는 Git bash이다. Git을 거의 처음 사용하다시피 하는거라 봐도 무방해서 앞으로 친숙해져야겠다는 생각이 든다.

streamlit과 Git 이용방법에 대해 많이 배웠다. 현업에서 팀프로젝트시에 Git bash를 이용하는 경우도 많다고 하니 익숙해져야겠다.

  - 앞으로 할 일(つぎにやること: 응용하여 배운 것을 어디에 어떻게 적용할지)

잠들어 있던 나의 기억들이 점점 깨어나고 있기에, 앞으로는 해당 기억들을 다시 효율적으로 쓸 수 있을 것 같다. 곧 다가올 첫 번째 팀프로젝트에 대비하여 배운 것들을 자유자재로 쓸 수 있도록 해야겠다.
